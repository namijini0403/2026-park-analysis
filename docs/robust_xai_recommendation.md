# Robust XAI Recommendation Layer

## 추가 이유

기존 후보 추천은 하나의 가중치 조합 또는 고정 우선순위에 가까워, 정책 담당자의 선호가 조금만 달라져도 추천 후보가 바뀌는지 확인하기 어렵다. 이번 레이어는 기존 후보지 생성, 기존 case 분류, 기존 AI 추천 흐름을 보존하면서 후보의 견고성을 별도 산출물로 덧붙인다.

## 단일 가중치 추천의 한계

단일 점수는 설명하기 쉽지만 수요, 접근성 보완성, 학교 근접성 중 무엇을 더 중요하게 볼지에 따라 결과가 민감할 수 있다. 특히 공공정책 의사결정에서는 하나의 정답 가중치를 선언하기보다, 다양한 정책 선호에서도 계속 상위권에 남는 후보를 우선 검토하는 편이 책임 있는 방식이다.

## 목적함수 3축

- `demand_score`: 2031 미래 예상 수혜 아동 수를 우선 사용하고, 없으면 2029 값을 사용한다. 높을수록 좋다.
- `access_gap_score`: 기존 활동가능공원 또는 최근접 공원과의 거리를 사용한다. 멀수록 기존 시설 보완 필요성이 크다고 해석한다. 이는 최종 정책 판단이 아니라 후보 보완성의 한 축이다.
- `proximity_score`: 학교와 후보지 간 경로거리 또는 후보 거리의 역정규화 점수다. 가까울수록 좋다.

정규화는 학교별 후보군 안에서 min-max scaling을 적용했다. 후보 수가 20개 이상인 그룹은 1-99 percentile clipping 후 min-max를 적용해 이상치 영향을 줄였다.

## Pareto 후보군

학교별 후보지 풀 안에서 `demand_score`, `access_gap_score`, `proximity_score` 세 축을 모두 비교한다. 어떤 후보 A가 후보 B보다 세 축 모두에서 같거나 좋고 적어도 하나에서 더 좋으면 A가 B를 지배한다. 다른 후보에게 지배당하지 않는 후보를 Pareto 후보로 표시한다.

## 1,000회 가중치 샘플링

정책 가중치는 하나의 정답으로 고정하지 않고, 1,000회 가중치 샘플링을 통해 다양한 정책 선호에서도 상위권에 안정적으로 남는 후보를 식별하였다.

샘플링은 `numpy.random.default_rng(seed=42)`와 `Dirichlet([1, 1, 1])`을 사용했다. 각 샘플에서 세 축 가중합 점수를 계산하고 학교별 순위를 산출했다. 후보별로 `top1_count`, `top3_count`, `top5_count`, `top5_stability_score`, `mean_rank`, `rank_std`, `best_rank`, `worst_rank`를 생성했다. 후보 수가 5개 미만인 학교는 전체 후보 중 상위권 안정성으로 계산한다.

## Robust Ranking

최종 robust 순위는 기존 정책 필터 통과 여부, Pareto 후보 여부, Top5 안정성, 평균 순위, 순위 표준편차, 기존 추천 점수 순서로 정렬한다. 이 순위는 기존 추천 로직을 삭제하지 않고 보강 필드로만 병합된다.

## SHAP의 역할

SHAP은 정책 가중치를 결정하지 않고, 견고성 분석을 통과한 후보의 미래 수요 예측 근거를 후보 단위로 분해한다. 따라서 SHAP은 자동추천의 근거가 아니라, human-in-the-loop 최종 판단을 위한 후보 진단 정보로 사용된다.

현재 프로젝트에는 후보 단위 미래 수요 예측값이 존재하지만, 배포 저장소에 원 학습 artifact와 feature matrix가 완전한 형태로 고정되어 있지는 않다. 따라서 이번 산출물은 기존 예측값을 설명하기 위한 surrogate explainability model을 사용했다. RandomForestRegressor로 `predicted_beneficiaries_used`를 설명하고, SHAP TreeExplainer로 후보별 양/음 기여 변수를 산출했다.

## SHAP이 하지 않는 것

SHAP은 최종 추천 순위를 설명하지 않는다. SHAP은 Pareto 여부, Top5 안정성, 평균 순위, 정책 필터를 대체하지 않는다. 또한 SHAP은 정책 가중치를 정하지 않으며, 특정 후보를 자동 확정하는 근거로 사용하지 않는다.

## HITL 의사결정 구조

AI는 미래 수요를 예측하고, SHAP은 후보별 예측 근거를 분해하며, 1,000회 가중치 샘플링은 정책 선호 변화에도 견고한 후보를 찾는다. 최종 판단은 SHAP 진단과 안정성 지표를 입력으로 받은 정책 담당자가 수행한다.

## 한계

- 공원 거리 축은 접근성 보완성을 단순화한 지표이며 현장 보행환경 전체를 대체하지 않는다.
- surrogate SHAP은 기존 예측값을 사후 설명하는 용도이므로 원 예측모델 자체의 완전한 내부 설명은 아니다.
- 도로 횡단, 사고위험, 재개발 변수는 공개데이터 기반 신호이므로 현장 검토가 필요하다.
- 후보가 적은 학교는 Top5 안정성이 전체 후보 안정성에 가깝게 해석된다.
