# 06 Limitations

### [chunk: 06_limitations#public-data]
tags: 공공데이터, 한계, 최신성, 누락, 기준시점, PPT기준

정의:
본 분석은 공개 가능한 공공데이터와 지도 기반 데이터를 사용한 정책 지원 분석이다.

해석:
최신 현장 상황, 임시 폐쇄, 공사, 실제 통학 동선, 주민 이용 행태, 공원 출입구, 미등록 놀이공간을 완전히 반영하지 못할 수 있다. 일부 데이터는 기준 시점이 다를 수 있으므로 정책 적용 전 최신 자료로 재검증해야 한다.

주의:
PPT 기준 어린이집·유치원 수요는 이번 분석의 수요 모델에 포함하지 않았다. 단지 내부 녹지나 미등록 놀이공간은 전수 확인이 어려워 공공자료 기준으로 보수적으로 해석한다.

잘못된 해석:
공공데이터 기반 분석 결과를 현장 실사 완료, 최신 행정 결정 반영, 실제 이용 행태 검증 완료로 설명하면 안 된다.

### [chunk: 06_limitations#walking-network]
tags: OSMnx, Valhalla, OSM, 보행네트워크, 아파트단지, 단지투과, PPT기준

정의:
보행 네트워크 한계는 OSM 도로망과 지도 데이터가 실제 보행 가능성을 완전하게 담지 못하는 문제를 뜻한다.

해석:
Case 판정과 도보권 녹지비율 산출 기준은 OSMnx 기반 도보권 지표를 따른다. 다만 앱의 지도 도보권 오버레이는 Valhalla 기반 파일을 보조 표시 레이어로 사용한다. OSM에 아파트 단지 내부 보행로, 사잇길, 출입구 정보가 누락되어 있으면 실제 보행 가능성과 차이가 날 수 있다.

주의:
PPT 기준 아파트 단지 내부 통행 가능성은 단지별 차이가 커서 본문 분석에서는 보수적으로 제외한다. 최종 해설에서는 본문 기준의 보수적 분석 결과만 설명한다.

잘못된 해석:
지도상 도보권이나 OSM 도로등급만으로 실제 보행 위험도, 단지 내부 통행 가능성, 학교 출입구 조건이 모두 검증됐다고 설명하면 안 된다.

### [chunk: 06_limitations#sealed-values]
tags: 봉인값, 수동검증, 재현성, 자동계산, 최근접공원거리, PPT기준

정의:
봉인값은 자동 산출값 중 오류 가능성이 큰 최근접 공원 도보거리를 수동 검증해 고정한 운영 기준값이다.

해석:
수동 검증으로 봉인한 값은 재현성과 발표 안정성을 위해 유지한다. 자동 재계산 결과와 다를 수 있지만, 봉인값은 Case 판정 중 거리 조건의 오분류를 줄이기 위한 검수된 운영 기준으로 해석한다.

주의:
봉인값은 최근접 공원 도보거리 검증값이며, 도보권 녹지비율이나 녹지면적 전체를 수동 재산출했다는 뜻이 아니다.

잘못된 해석:
봉인값이 있다고 해서 녹지면적, 공원 기능, 설치 가능성, 안전성, 현장 이용성이 모두 검증됐다고 설명하면 안 된다.

### [chunk: 06_limitations#field-review]
tags: 현장검토, 안전, 토지이용, 설치가능성, 250m격자, 후보지, PPT기준

정의:
후보지 추천은 정책 검토 대상을 좁히는 기능이며, 설치 가능성을 확정하는 기능이 아니다.

해석:
250m 후보지는 정확한 설치 좌표가 아니라 검토 블록이다. 실제 설치 여부는 토지 소유, 지장물, 안전, 민원, 예산, 유지관리 책임, 현장 보행환경, 도시계획 조건을 정책 담당자가 확인해야 한다.

주의:
AI 추천은 출발점이고 최종 판단은 사람이 수행한다. 안전·재개발·대단지 변수는 최종 점수 안에 숨기지 않고 필터와 확인사항으로 공개한다.

잘못된 해석:
추천 후보지를 확정 설치 위치, 토지 확보 완료 지점, 공사 가능성이 검증된 위치로 설명하면 안 된다.

### [chunk: 06_limitations#llm]
tags: LLM, RAG, 근거문서, 답변제한, 비식별, context_pack, PPT기준

정의:
LLM 해설은 선택된 학교·후보지 context pack과 검색된 근거 문서 안에서만 설명하는 RAG-lite 보조 기능이다.

해석:
근거 문서에 없는 법적 판단, 예산 산정, 실제 공사 가능성, 최신 행정 결정, 데이터 밖 신규 추천은 답변할 수 없다. 선택된 context가 없으면 개별 학교·후보지 수치 해설을 하면 안 된다.

주의:
비식별 공개 모드에서는 특정 학교명, 후보지 코드, 공원명, 지역명, 좌표, 거리값, 수혜인원 값을 답변하지 않고 일반 개념 설명으로 전환해야 한다. 식별 제한 접근 모드에서도 근거 chunk와 선택 데이터에 없는 값을 만들어내면 안 된다.

잘못된 해석:
LLM 해설을 정책 확정, 법적 자문, 예산 산정, 최신 행정 상황 조회, 데이터베이스 직접 검색 기능으로 설명하면 안 된다.
