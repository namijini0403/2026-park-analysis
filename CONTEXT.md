## 핵심 분석 구조

### 전체 흐름
```
절대값 → 상대값 → 행정-물리 불일치 → 유형화 → 우선순위
```

### 레이어 구조 (2×2)
| | 학구도 근사(Voronoi) | 500m 반경 |
|--|---------------------|-----------|
| **직선거리** | case 분류 보조 | 비교용 |
| **실제 도보거리** | 핵심 분석 | 핵심 분석 |

→ 앱에서 두 레이어 동시 표시 (직선 원 vs 도보 등시선 비교)

### case_type 분류
- **case 1**: 학구도 내 + 도보 500m 내 → 양호
- **case 2**: 학구도 외 + 도보 500m 내 → 외부 의존형
- **case 3**: 학구도 내 + 도보 500m 외 → 행정 착시 (핵심 분석 포인트)
- **case 4**: 둘 다 외 → 절대 취약

## 변수 구조

### 기본 환경 변수
```
park_count_500m_straight     # 직선 500m 내 공원 수 (비교용)
park_count_500m_walk         # 실제 도보 500m 내 공원 수 (핵심)

playground_count_500m_walk   # 실제 도보 500m 내 놀이터 수

detour_extra_avg             # 우회 시 추가되는 평균 거리(m)
barrier_flag                 # 단절 요소 존재 여부 (True/False)
barrier_type                 # 단절 요소 종류 (철로/대로/하천)
case_type                    # 1~4 분류
```

### 상대값 변수
```
expected_park                # 유사 규모 집단 평균 기반 기대값
                             # 보조: 도시공원법 기준 (어린이 500명당 1개소)
gap                          # actual - expected (음수 = 부족)
gap_category                 # 상대적취약 / 보통 / 양호
```

### 수요 변수
```
student_count                # 학교별 학생 수
child_pop_0_5                # 반경 내 영유아 (0~5세) - 수혜인원 보정용

forecast_3y                  # 3년 후 수혜인원 예측
forecast_5y                  # 5년 후 수혜인원 예측
forecast_10y_scenario        # 10년 시나리오 (방향성만, 정밀예측 아님)
```

### 정책 레이어 변수 (후처리, 기본 분석과 분리)
```
redevelopment_flag           # 재개발 구역 여부
redevelopment_year           # 재개발 예정 연도
vulnerable_child_ratio       # 취약계층 아동 비율
```

## 분석 파이프라인

### PHASE 1: 데이터 전처리
- 파일 인코딩 정제 (CP949 → UTF-8)
- 컬럼명 영문 통일
- 결측값·중복값 제거
- 좌표계 통일 (WGS84)
- 인천 지역 필터링 (전국 단위 파일에서)
- 지오코딩 필요 시: 주소 → 위경도 변환
- 지오코딩 실패 목록 자동 출력 → 사람 검수
- 100m 격자 인구에서 인천 영역만 추출
- 공원 종류 필터링: 어린이공원·소공원·근린공원·수변공원만 포함
  (완충녹지·경관녹지·연결녹지 제외)
- 수변공간 유효성: 하천 buffer ∩ 공원 영역 겹치는 경우만 인정

### PHASE 2: 공간 분석
- OSMnx로 인천 보행 도로망 자동 수집
- 철로·대로·하천 장벽 레이어 설정
- 학교별 직선 500m 원 생성 (비교용)
- 학교별 실제 도보 500m 등시선(isochrone) 생성 (핵심)
- 학교-공원 매핑 (직선거리 + 실제 도보거리 둘 다 계산)
- 우회 추가거리 계산 (detour_extra)
- barrier_flag 및 barrier_type 설정
- Voronoi 다이어그램으로 근사 학구도 생성
- case_type 분류 (1~4)

### PHASE 3: 상대값 분석
- expected_park 계산 (유사 규모 집단 평균)
- 보조 기준: 도시공원법 (어린이 500명당 1개소)
- gap 산출 및 gap_category 분류

### PHASE 4: 클러스터링 + 이상치 탐지
- 입력 변수: gap, case_type, barrier_flag, detour_extra_avg
- K-means 클러스터링 (군집 수는 결과 보고 결정)
- Isolation Forest 이상치 탐지
- 군집 유형 명칭 부여는 반드시 사람이 판단
- 예상 군집 유형:
  - A: 양호형
  - B: 행정OK-실제단절형 (gap<0, case 3)
  - C: 외부의존형 (gap<0, case 2)
  - D: 행정착시+단절 이중고형 (gap<0, case 3 + barrier)
  - E: 절대취약형 (gap<<0, case 4)
- 부정 이상치(주변 양호한데 유독 취약) → 우선순위 핵심 근거

### PHASE 5: 인구 예측
- 코호트 분석 중심 (2009~2024 학생수 추이 기반)
- Prophet / ARIMA 비교 보조
- 재개발 변수 보정 적용
- 예측 범위:
  - 현재: 기본
  - 3년: 핵심
  - 5년: 보조
  - 10년: 시나리오만 (정밀예측 아님, 발표에 반드시 명시)

### PHASE 6: 우선순위 도출
- 가중치 합산 점수 방식 절대 사용 금지
- 규칙 기반 필터 방식
```
1순위: gap < 0 AND case_type == 3 AND barrier_flag == True
2순위: gap < 0 AND case_type == 3
3순위: gap < 0 AND barrier_flag == True
4순위: gap < 0
```
- **놀이터 변수 역할 분리 확정**
- 공공 놀이터 (parks.csv) = 행정 등록 데이터, "없음=없음" 성립
- OSM 민간 놀이터 = 참고 지표, 보조분석/민감도 확인용으로만 사용
- 한 줄 원칙: 공공은 해석용, 민간은 참고용
- 우선순위 기준 변경 시 반드시 사용자와 먼저 협의

## AI 모델 구조 (2026-04-07 확정)
공모전명: 교육공공데이터 AI활용 공모전

### [AI 사용처 3개]

#### 1. 클러스터링 (완료)
- k-means k=3, 실루엣 0.4156
- cluster0 접근양호형 / cluster1 도보취약형 / cluster2 대형공원특수형
- 이상치 14개 (Isolation Forest)

#### 2. 수혜인구 예측 (미완)
- 모델: XGBoost 회귀
- 타겟: 학교별 등시선 내 0~12세 아동인구
- 피처: 격자 총인구, 재개발 상태, 구 더미, 등시선 내 시설 수, 도로망 밀도
- SHAP으로 피처 중요도 시각화
- 예측 연도: 현재 / 2029 / 2031
- 출력: data_processed/beneficiary_forecast.csv
  컬럼: 학교명, gu, dong, current_beneficiary,
        forecast_2029, forecast_2031,
        redev_flag, confidence_low, confidence_high

#### 3. 우선지원 학교 선정 보완 (미완)
- 모델: LightGBM 분류
- 타겟: 우선지원 여부 (현재 rule-based priority_score를 레이블로)
- 피처: access_ratio, gap_count, child_pop_quartile,
        redev_status, cluster, iso_park_count
- rule-based vs ML 결과 불일치 케이스 별도 분석
- 출력: data_processed/priority_ml.csv

### [방법론 한계]
- 데이터 272개 소규모 → 과적합 위험
  대응: cross-validation + 피처 수 제한
- 동 단위 출생 시계열 미존재 (개인정보·표본 크기 제약)
  → 격자 아동인구 + 학생수 시계열로 대체
  → 단기 예측(~5년) 범위에서 출생 데이터 기여 제한적 판단
- OSM 기반 민간 놀이터는 참고 지표이며,
  누락이 체계적으로 발생할 수 있어 핵심 해석에 사용하지 않음

  (개교 연도 이전 데이터 부재, 분석에서 제외 또는 보유 연도만 사용)

## 앱 (app/index.html) 업데이트 (2026-04-07)
- 범위: 인천 전체 272개교
- 초기 줌레벨: 10 (인천 전체 보이는 수준)
- 타이틀: "인천시 초등학교 야외활동 환경 격차 지도"

### 지도 레이어 (토글 가능)
```
□ 초등학교 마커
□ 공원·놀이터 마커
□ 직선 반경 500m (반투명 파란 원) ← 비교용
□ 실제 도보 반경 500m 등시선 (반투명 초록) ← 핵심
□ 단절 요소 선 (철로/대로/하천)
□ 인구 히트맵
□ 재개발 구역
□ 우선지원 지역 하이라이트
```

### 구 드롭다운
- 전체 + 인천 10개 구/군
- 선택 시: 해당 구 우선지원 마커 하이라이트 + 자동 줌 이동
- 줌 이동 방식: 해당 구 학교들의 centroid 계산 (별도 경계 geojson 불필요)

### 좌측 패널 (구 선택 시 요약 통계)
```
- 총 학교 수
- case_type 분포 (1/2/3/4 각 몇 개)
- 평균 access_ratio
- 우선지원 목록 (priority_score 3점 이상)
- 등시선 내 공원 0개 학교 수
```

### 우선지원 마커
- priority_score 3~4점 → 깜빡임 또는 강조

### 학교 클릭 시 상세 패널
```
- 학교명 / 상태 / 군집 유형
- 직선 500m 내 공원 수 vs 도보 500m 내 공원 수
- gap 수치 (유사조건 대비 ±N개)
- case_type 및 설명
- 단절 요소 종류 / 우회 평균 추가거리
- 현재 수혜인원 / 3년 후 / 5년 후
- 이상치 여부 및 설명 문장
```

### 시뮬레이션 기능
```
- 후보지 선택 (지도 클릭 또는 드롭다운)
- 시설 유형 선택: 놀이터 / 소공원 / 수변공원
- 연도 슬라이더: 현재 / 3년 후 / 5년 후
- 단절 요소 경고 표시
- 예상 수혜인원 변화 + 영향 학교 목록
```

### population_grid_1k.csv 아동인구 보정 (2026-04-XX)

**배경**
SGIS 1K 격자 인구는 5세 구간 단위 제공.
in_age_001=0-4세, in_age_002=5-9세, in_age_003=10-14세
→ 목표 구간(0-5세 영유아 / 6-12세 초등학령기)과 불일치.

**보정 방법: 시군구 1세별 비율 안분**
`성및연령별_추계인구_1세별.xlsx` (인천광역시 전체) 사용.
5세 구간 내 각 연령의 구성비를 인천 시 단위 평균으로 산출 후
격자별 5세 구간 인구에 비율 적용하여 1세별 추정치 생성.

child_pop_0_5  = 추정 0세+1세+2세+3세+4세+5세
child_pop_6_12 = 추정 6세+7세+8세+9세+10세+11세+12세

**한계 명시 (보고서 방법론 섹션 필수 기재)**
- 격자 단위 연령 구조가 인천 평균과 다를 수 있음
- 재개발/신도시 격자는 실제 연령 구조 편차 가능성 높음
- 안분 비율은 2024년 추계인구 기준 고정값 (격자별 변동 미반영)
- 영향 범위: child_pop 절대값 해석 시 ±10% 내외 오차 가능
  상대 비교(격자 간 격차 분석)에는 영향 미미

**검증 파일**
data_processed/age_ratio_incheon.csv

### 새 분류체계
- `is_separate_bundle_tag = 1`
  - `gu == 강화군`
  - 또는 `gu == 옹진군`
  - 또는 학교명이 `분교`, `분교장` 포함
- 별도 묶음 대상은 `case1~4` 분포 집계에서 제외하고 별도 관리한다.

- `is_low_access_tag = 1`
  - `iso_park_count >= 1`
  - 그리고 `access_ratio <= 0.5`
- 의미: 도보권 공원은 최소 1개 있으나, 직선 500m 내 공원 대비 실제 보행 접근성이 낮다.

- `is_case_conflict_tag = 1`
  - `nearest_park_dist_m < 500` 이지만 `iso_park_count == 0`
  - 또는 `nearest_park_dist_m >= 500` 이지만 `iso_park_count >= 1`
- 운영 규칙: 충돌 케이스는 임시로 `case2`에 편입하고, 이후 실측 검수로 재보정한다.

### case 정의
- `case1`
  - `nearest_park_dist_m >= 500`
  - 그리고 `iso_park_count == 0`
- `case2`
  - `nearest_park_dist_m < 500`
  - 그리고 `iso_park_count >= 1`
  - 그리고 `iso_green_ratio` 하위 1/3
- `case3`
  - `nearest_park_dist_m < 500`
  - 그리고 `iso_park_count >= 1`
  - 그리고 `iso_green_ratio` 중간 1/3
- `case4`
  - `nearest_park_dist_m < 500`
  - 그리고 `iso_park_count >= 1`
  - 그리고 `iso_green_ratio` 상위 1/3

### 산출 원칙
- `iso_park_count`, `iso_park_area`는 `시설유형 != 놀이터` 공공공원 기준으로 재산출한다.
- `iso_green_ratio = iso_park_area / isochrone_area_m2 * 100`
